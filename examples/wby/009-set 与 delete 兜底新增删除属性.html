⑨ $set / $delete 兜底新增删除属性

我们的响应式机制依赖"在 observe 时给已有的 key 加上 getter/setter"。
那么——
  • 之后新增的 key 上没有 setter，赋值不会通知；
  • delete 一个 key，更没有谁通知。
Vue 的解法是提供 Vue.set 和 Vue.delete（实例上是 $set/$delete）作为显式兜底通道：
内部主动 defineReactive 新键，并通过对象级的 __ob__.dep 通知更新。

③ 日志

关键 1：$set 的核心三步
  1. 已经在对象上：直接赋值（getter/setter 已生效）；
  2. 是数组 + 合法索引：target.splice(key, 1, val)——借 008 重写过的 splice 自动通知；
  3. 是新键：defineReactive(ob.value, key, val)，再调 ob.dep.notify()。

关键 2：__ob__ 的双重身份
  • 作为 "已经被 observe 过"的标记——$set 的入参如果没有 __ob__，意味着它本来就不是响应式的，直接赋值就行；
  • 作为 对象级的 dep 持有者——新增 key 时没有"属性级 dep"可以通知，必须借 ob.dep 这个"对象整体的 dep"。
    那为什么 watcher 会订阅 ob.dep 呢？回看 008 的 getter：if (childOb) childOb.dep.depend() 这一行就是在做这件事。
遗留问题：到现在为止，每改一次数据都会立刻触发一次完整的 render。

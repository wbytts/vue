⑬ deep watch（traverse 递归触发深层 getter）

我们的 watcher 默认只对"fn 里直接读到的属性"建立依赖。
那么这种写法：
  $watch(() => data.config, cb)
fn 只读了一次 config（外层），没有读 config 的子属性，
所以改 config.theme 时，cb 不会触发——只有把 config 整个替换才会。

Vue 的解法是 traverse：
  在 get() 完成后，如果 watcher 开启了 deep: true，就
  递归"摸一遍"返回值的所有嵌套属性——每次 ".xxx" 触发的 getter 都会让
  当前 Dep.target 和这个嵌套的 dep 建立订阅。
这相当于"主动手工帮你读一遍每个 key"。

② 操作区 deep OFF

步骤建议：
1️⃣ 先在 deep OFF 下，点"改 theme/locale/嵌套"——cb 不会触发，因为 watcher 只依赖 config 这一层；
2️⃣ 切到 deep ON，再点同样的按钮——cb 触发了；
3️⃣ 在两种模式下，"替换整个 config 对象"都能触发 cb（因为外层引用变了）。

③ 日志

traverse 的核心：
  1. 是数组就遍历每一项；是对象就遍历每个 key；递归下去；
  2. 用一个 seen 集合记录已访问过的 dep id，避免循环引用导致死循环；
  3. 跳过冻结对象 / 已标记跳过的对象 / VNode 等。

注意：traverse 是个性能"重操作"——每次重新求值都会把整个对象树读一遍。
在大对象上滥用 deep: true 会有可见的开销。Vue 3 的 watch 也保留了类似的开关。
$watch 与 render-watcher 的差异：
  • render-watcher 的 fn 自然会读到模板里所有用到的属性，依赖收集是自动且完备的；
  • 但 $watch('config', cb) 这种 user watcher 的 fn 只是 () => data.config，
    只能读到一层——deep: true 就是为这种场景设计的。
